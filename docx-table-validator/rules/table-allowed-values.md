## 值域限制檢查

**嚴重程度:** ERROR

確保特定欄位的值在允許的值域範圍內。非標準值可能導致資料不一致。

### 目標識別

**表格匹配條件：**

```yaml
matcher:
  type: column-headers
  columns:
    - 影響程度
    - 發生機率
```

### 驗證邏輯

檢查指定欄位的值是否在允許列表中。

**欄位: 影響程度**
允許值：

- 高
- 中
- 低

**欄位: 發生機率**
允許值：

- 高
- 中
- 低

**錯誤範例：**

```
| 風險編號 | 影響程度 | 發生機率 |
|----------|----------|----------|
| R001     | 嚴重     | 很高     |  ← "嚴重"、"很高" 不在允許值中
```

**正確範例：**

```
| 風險編號 | 影響程度 | 發生機率 |
|----------|----------|----------|
| R001     | 高       | 高       |
```

### 例外情況

空值不觸發此規則（由 `table-required-fields` 處理）。
