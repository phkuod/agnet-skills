## 資料行完整性檢查

**嚴重程度:** ERROR

當表格的第一欄位有值時，表示該行是有效的資料行，後續所有欄位都不可為空。

### 目標識別

**表格匹配條件：**

```yaml
matcher:
  type: any-table
  # 此規則適用於所有表格
```

### 驗證邏輯

**規則：** 若 row[第一欄] 有值，則 row[其他所有欄位] 都必須有值。

**檢查流程：**

1. 檢查每一行的第一個欄位
2. 若第一欄有值（非空）
3. 則檢查該行的所有其他欄位
4. 若有任何欄位為空，標記為錯誤

**錯誤範例：**

```
| 編號 | 名稱   | 數量 | 備註 |
|------|--------|------|------|
| 001  | 產品A  | 100  | 正常 |
| 002  | 產品B  |      |      |  ← 錯誤：編號有值，但數量和備註為空
| 003  |        | 50   | 測試 |  ← 錯誤：編號有值，但名稱為空
```

**正確範例：**

```
| 編號 | 名稱   | 數量 | 備註 |
|------|--------|------|------|
| 001  | 產品A  | 100  | 正常 |
| 002  | 產品B  | 200  | 正常 |
|      |        |      |      |  ← 正確：第一欄為空，整行可為空
```

**另一個正確範例（允許整行為空）：**

```
| 編號 | 名稱   | 數量 |
|------|--------|------|
| 001  | 產品A  | 100  |
|      |        |      |  ← 正確：第一欄為空時不檢查
| 003  | 產品C  | 300  |
```

### 例外情況

- 第一欄為空的行不觸發此規則（視為空白行或分隔行）
- 表頭行不參與驗證
