## 條件必填檢查

**嚴重程度:** ERROR

當某欄位符合特定條件時，另一欄位必須有值。這確保相關資料的完整性。

### 目標識別

**表格匹配條件：**

```yaml
matcher:
  type: column-headers
  columns:
    - 影響程度
    - 緩解措施
```

### 驗證邏輯

**條件規則：**

- 當「影響程度」= 「高」時，「緩解措施」不可為空

**錯誤範例：**

```
| 風險編號 | 影響程度 | 緩解措施 |
|----------|----------|----------|
| R001     | 高       |          |  ← 影響程度為高，但緩解措施為空
```

**正確範例：**

```
| 風險編號 | 影響程度 | 緩解措施     |
|----------|----------|--------------|
| R001     | 高       | 增加備援系統 |
| R002     | 低       |              |  ← 影響程度為低，緩解措施可為空
```

### 例外情況

當觸發條件欄位為空時，不觸發此規則。
